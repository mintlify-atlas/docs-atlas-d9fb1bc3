## HTTP endpoints

### GET /webhook

Webhook verification endpoint. Meta's Cloud API sends a `GET` request to this URL when you register or update the webhook in the Meta Developer Console. Yoko responds with the `hub.challenge` value to confirm ownership of the endpoint.

#### Query parameters

<ParamField query="hub.verify_token" type="string" required>
  The token sent by Meta to verify your endpoint. Must exactly match `VERIFY_TOKEN` in `app.py` (case-sensitive).
</ParamField>

<ParamField query="hub.challenge" type="string" required>
  A random string generated by Meta. Yoko echoes this value back as the response body on success.
</ParamField>

<ParamField query="hub.mode" type="string">
  Always `"subscribe"` when sent by Meta. Not validated by Yoko.
</ParamField>

#### Responses

| Status | Body | Condition |
| --- | --- | --- |
| `200 OK` | The value of `hub.challenge` as plain text | `hub.verify_token` matches `VERIFY_TOKEN` |
| `403 Forbidden` | `Token inválido` | Token mismatch |

#### Example

```bash
curl "http://localhost:5000/webhook?hub.verify_token=YOUR_TOKEN&hub.challenge=1234567890"
```

### POST /webhook

Receives incoming WhatsApp messages forwarded by Meta's Cloud API. Yoko extracts the message text, generates a response via the LLM pipeline, and sends the reply back to the sender.

<Note>
  Yoko always returns `200 OK` regardless of whether processing succeeded. This prevents Meta from retrying the delivery. Errors are printed to stdout via `traceback.print_exc()`.
</Note>

#### Request body

Standard WhatsApp Cloud API webhook payload:

<ParamField body="entry" type="object[]" required>
  Top-level array of change entries from Meta.

  <Expandable title="entry[0].changes[0].value">
    <ParamField body="messages" type="object[]">
      Present when the event is an inbound message. Absent for `statuses` events (delivery receipts, read receipts), which Yoko silently ignores.

      <Expandable title="messages[0]">
        <ParamField body="from" type="string" required>
          Sender's WhatsApp phone number including country code, e.g. `"5491112345678"`.
        </ParamField>
        <ParamField body="text" type="object" required>
          Present only for `type: "text"` messages. Messages of other types (image, audio, etc.) will cause a `KeyError` and be caught by the exception handler.
          <Expandable title="text">
            <ParamField body="body" type="string" required>
              The raw message text entered by the user.
            </ParamField>
          </Expandable>
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

#### Expected payload structure

```json
{
  "entry": [
    {
      "changes": [
        {
          "value": {
            "messages": [
              {
                "from": "5491112345678",
                "type": "text",
                "text": {
                  "body": "¿Cuándo es el examen de cálculo?"
                }
              }
            ]
          }
        }
      ]
    }
  ]
}
```

#### Message path extraction

The message fields are accessed via:

```python
entry = data["entry"][0]["changes"][0]["value"]
mensaje = entry["messages"][0]["text"]["body"]
numero  = entry["messages"][0]["from"]
```

#### Responses

| Status | Body | Condition |
| --- | --- | --- |
| `200 OK` | `OK` | Always — including on processing errors |

## Internal Python functions

These functions are not HTTP endpoints but are the core of Yoko's processing pipeline.

### `manejar_query(mensaje_usuario)`

Defined in `mcp.py`. Entry point for processing a WhatsApp message.

**Parameters**

<ParamField path="mensaje_usuario" type="str" required>
  The raw text received from the WhatsApp message body.
</ParamField>

**Returns:** `str` — the LLM-generated reply to send back to the user.

**Behavior:** Calls `buscar_contexto` to retrieve relevant context from `data.json`, then passes that context and the original question to `consultar_llm`.

---

### `buscar_contexto(pregunta_usuario)`

Defined in `mcp.py`. Searches `data.json` for the closest matching question.

**Parameters**

<ParamField path="pregunta_usuario" type="str" required>
  The user's question text.
</ParamField>

**Returns:** `str` — the `respuesta` field from the best-matching entry in `data.json`, or `"No se encontró información relevante."` if no match meets the threshold.

**Behavior:** Loads `data.json` on every call and uses `difflib.get_close_matches` with `n=1, cutoff=0.4` to find the nearest question.

<Tip>
  To tune matching sensitivity, adjust the `cutoff` parameter. Lower values (e.g. `0.2`) are more permissive; higher values (e.g. `0.6`) require closer matches. See the [FAQ](/reference/faq) for details.
</Tip>

---

### `consultar_llm(contexto, pregunta)`

Defined in `llm.py`. Sends a chat completion request to the Groq API.

**Parameters**

<ParamField path="contexto" type="str" required>
  The context string retrieved from `data.json` (or the fallback message if nothing was found).
</ParamField>

<ParamField path="pregunta" type="str" required>
  The original user question.
</ParamField>

**Returns:** `str` — the model's response text, or `"No se pudo generar una respuesta."` if the API response does not contain a `choices` field.

**Details:**
- API endpoint: `https://api.groq.com/openai/v1/chat/completions`
- Model: `llama3-8b-8192`
- Temperature: `0.3`
- System prompt: `"Eres un asistente universitario responde usando solo esta información: {contexto}"`

---

### `enviar_mensaje(numero, texto)`

Defined in `app.py`. Sends a text message to a WhatsApp number via the Cloud API.

**Parameters**

<ParamField path="numero" type="str" required>
  The recipient's WhatsApp phone number including country code (e.g. `"5491112345678"`).
</ParamField>

<ParamField path="texto" type="str" required>
  The message text to send.
</ParamField>

**Returns:** None. The HTTP response from the WhatsApp API is not currently checked.

**Payload sent to WhatsApp Cloud API:**

```json
{
  "messaging_product": "whatsapp",
  "to": "5491112345678",
  "type": "text",
  "text": {
    "body": "El examen de cálculo es el viernes 20 a las 10h."
  }
}
```
