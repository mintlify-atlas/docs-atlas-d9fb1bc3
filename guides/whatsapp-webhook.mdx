---
title: "WhatsApp webhook setup"
description: "Connect Meta's Cloud API to Yoko"
---

Meta's WhatsApp Cloud API delivers incoming messages to your app by sending HTTP POST requests to a URL you provide. That URL must be:

- Publicly accessible over the internet
- Served over HTTPS
- Responding correctly to a one-time GET verification request before messages are forwarded

During development, your Flask server runs on `localhost` and is not reachable by Meta. The steps below cover how to expose it temporarily with a tunnel and then wire it up in the Meta Developer Console.

<Steps>
  <Step title="Expose localhost with ngrok">
    Install [ngrok](https://ngrok.com) if you haven't already, then run:

    ```bash
    ngrok http 5000
    ```

    ngrok prints a forwarding URL like `https://xxxx.ngrok.io`. Copy the HTTPS address — you'll use it in the next step.

    <Warning>
      Free ngrok accounts receive a new random URL every time you restart the tunnel. Any time the URL changes you must update the Callback URL in the Meta Developer Console. Use a paid ngrok plan or a proper deployment (see [Deploying](/guides/deploying)) to avoid this in production.
    </Warning>
  </Step>

  <Step title="Open the Meta Developer Console">
    Go to [developers.facebook.com](https://developers.facebook.com), select your app, then navigate to **WhatsApp** → **Configuration** in the left sidebar.
  </Step>

  <Step title="Set the Callback URL">
    In the **Webhook** section, enter your ngrok URL with the `/webhook` path:

    ```
    https://xxxx.ngrok.io/webhook
    ```

    Replace `xxxx.ngrok.io` with your actual ngrok subdomain.
  </Step>

  <Step title="Set the Verify Token">
    Enter the same value you assigned to `VERIFY_TOKEN` in `app.py`.

    ```python
    VERIFY_TOKEN = "my_secret_token"  # must match what you type in the console
    ```
  </Step>

  <Step title="Click Verify and Save">
    When you click **Verify and Save**, Meta sends a GET request to your `/webhook` endpoint:

    ```
    GET /webhook?hub.mode=subscribe&hub.verify_token=YOUR_TOKEN&hub.challenge=1234567890
    ```

    Yoko's webhook handler compares `hub.verify_token` against `VERIFY_TOKEN`. If they match, it returns the `hub.challenge` value as plain text with a `200` status:

    ```
    1234567890
    ```

    Meta confirms the URL as valid and the configuration is saved.

    <Note>
      The `hub.challenge` must be returned as plain text, not JSON. Returning `{"challenge": "1234567890"}` will cause verification to fail.
    </Note>
  </Step>

  <Step title="Subscribe to the messages field">
    Still in the **Configuration** page, scroll to **Webhook Fields** and enable the **messages** field. This tells Meta to forward incoming WhatsApp messages to your webhook.
  </Step>

  <Step title="Send a test message">
    Send a WhatsApp message to your registered phone number. Watch the Flask console — you should see the incoming JSON payload printed (or a traceback if something is misconfigured). If the message reaches Yoko, you'll receive a reply generated by the LLM.
  </Step>
</Steps>
