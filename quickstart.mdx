This guide walks you through cloning the repository, installing dependencies, creating your knowledge base, and sending your first message through Yoko.

## Prerequisites

- Python 3.8 or higher
- A [Groq API key](https://console.groq.com) (free tier available)
- A Meta Developer account with WhatsApp Cloud API access

## Set up Yoko

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/juliorbk/yoko.git
    cd yoko
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    <Tabs>
      <Tab title="macOS / Linux">
        ```bash
        python3 -m venv venv
        source venv/bin/activate
        ```
      </Tab>
      <Tab title="Windows">
        ```bash
        python -m venv venv
        venv\Scripts\activate
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install flask requests
    ```

    <Note>
      There is no `requirements.txt` in the repository yet. The two required packages are `flask` (web server) and `requests` (HTTP client for Groq and WhatsApp APIs).
    </Note>
  </Step>

  <Step title="Create your knowledge base">
    Create a `data.json` file in the project root with your university FAQ entries. Each entry has a `pregunta` (question) and a `respuesta` (answer):

    ```json data.json
    [
      {
        "pregunta": "¿Cuáles son los requisitos para inscribirse?",
        "respuesta": "Para inscribirse necesitas tu cédula de identidad vigente, una foto tipo carnet reciente y el comprobante de pago de los aranceles de inscripción."
      },
      {
        "pregunta": "¿Cuándo comienzan las clases del semestre?",
        "respuesta": "Las clases del período 2025-I comienzan el 17 de febrero y finalizan el 30 de junio de 2025."
      },
      {
        "pregunta": "¿Dónde está la oficina de control de estudios?",
        "respuesta": "La oficina de Control de Estudios se encuentra en el edificio administrativo, planta baja, ventanilla 3. Horario: lunes a viernes de 8:00 a.m. a 12:00 p.m."
      }
    ]
    ```
  </Step>

  <Step title="Configure your API credentials">
    Open `llm.py` and set your Groq API key:

    ```python llm.py
    GROQ_API_KEY = "gsk_xxxxxxxxxxxxxxxxxxxxxxxx"  # your Groq API key
    GROQ_MODEL = "llama3-8b-8192"
    ```

    Open `app.py` and set your WhatsApp credentials and verify token:

    ```python app.py
    VERIFY_TOKEN = "my_secret_verify_token"  # choose any string

    # Inside enviar_mensaje():
    url = "https://graph.facebook.com/v17.0/747XXXXXXXXX/messages"
    headers = {
        "Authorization": "Bearer EAAxxxxxxxxxxxxxxxx",  # your WhatsApp token
        "Content-Type": "application/json"
    }
    ```

    <Warning>
      Never commit API keys or tokens to version control. Move them to a `.env` file using [python-dotenv](https://pypi.org/project/python-dotenv/) before pushing to a repository. See the [configuration guide](/guides/configuration) for details.
    </Warning>
  </Step>

  <Step title="Start the Flask server">
    ```bash
    python app.py
    ```

    You should see:
    ```
     * Running on http://127.0.0.1:5000
     * Debug mode: on
    ```
  </Step>

  <Step title="Test webhook verification">
    In a separate terminal, send a GET request simulating Meta's webhook verification:

    ```bash
    curl "http://localhost:5000/webhook?hub.verify_token=my_secret_verify_token&hub.challenge=1234567890"
    ```

    Expected response: `1234567890` (the challenge value echoed back as plain text with HTTP 200).
  </Step>
</Steps>
